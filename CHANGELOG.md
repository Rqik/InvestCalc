# Changelog

## 2026-04-24
- Дочищена типизация после рефакторинга: `*.types.ts` стали основным источником props/types, а битая русская кодировка в интерфейсе исправлена.
- Архитектурный рефакторинг: компоненты, UI primitives, pages и layout разнесены по папкам с *.types.ts, *.module.scss и index.ts; импорты переведены на @/.
- Стили приведены ближе к компонентному BEM: общие `metric-card`, `wait-card`, `summary-chip` и `help-card` убраны из компонентов в пользу собственных block-names и Sass-mixins для повторяющихся паттернов.
- Адаптивный слой разнесен по самим Sass-блокам через общие media-mixins, а центральный `_responsive.scss` убран, чтобы брейкпоинты жили рядом с конкретными компонентами.
- Для адаптива добавлены общие Sass-миксины брейкпоинтов, а порог выхода из desktop-shell поднят, чтобы средние ширины раньше переходили в более безопасный stacked-layout без срезанного контента.
- Адаптив пересобран по слоям: на планшетах fixed sidebar убран, навигация переехала внутрь контента, а на телефонах годовой план теперь открывается карточками вместо широкой таблицы.
- Убраны основные mobile-overflow проблемы: шапка, кнопки, легенда графика, help-карточки и таблица стали безопаснее по ширине и больше не должны выталкивать экран.
- Адаптивность пересобрана для планшетов и мобильных: форма, график, help-блоки, сценарии и пенсионные секции теперь раньше складываются в одну колонку и читаются устойчивее на узких экранах.
- Исправлена Sass-ошибка production-сборки: золотой shimmer в блоке `Если подождать еще` больше не зависит от межфайлового `@extend`.
- Золотой shimmer вынесен в отдельный Sass-модуль с миксином, а дублирующаяся анимация и рамка убраны из `metrics` и `extra-years`.
- Темная тема подправлена по акцентам: статусные карточки, блок `Если подождать еще` и график получили более глубокие фоны и чище читаемые цвета.
- Добавлены SVG-логотип, favicon и отдельный meta-компонент, который обновляет title и описание для калькулятора и пенсионной страницы.
- В tooltip графика убраны дублирующиеся подписи: теперь для каждой метрики показывается только одна строка вместо пары `dataKey` и человекочитаемого названия.
- В блоке сценариев счетчик перенесен ближе к заголовку секции, а внутренний scrollbar списка сделан тоньше и спокойнее визуально.
- Нижние help-блоки калькулятора стали полезнее: методика теперь объясняет текущий план живыми данными, а «Примеры» превращены в сравнение сценариев с явным применением в форму.
- Блок «Как читается ваш расчет» переработан в более наглядные карточки с нейтральной типографикой без цветных заголовков текста.
- График стал управляемым: можно скрывать отдельные линии, смотреть ряд ежемесячного взноса и переключать отображение в режим сегодняшних денег.
- Панель управления графиком стала компактнее: режим отображения теперь оформлен как отдельный аккуратный control-блок, а не как растянутая кнопка на всю ширину.
- На графике при наведении добавлена направляющая линия, чтобы было проще соотносить tooltip с уровнем значений.
- На графике hover-направляющая стала двусторонней, а линия «Вложено» в режиме сегодняшних денег теперь считается как реальная накопленная сумма без визуального убывания от инфляции.
- При сценарии `инфляция > доходности` в методике теперь показывается отдельный заметный совет, а формулировка про прибыль от роста сделана понятнее и без двусмысленности.
- Если тема не сохранена, приложение снова выбирает светлую или темную тему по системной настройке устройства.
- Исправлен путь к SVG-логотипу и favicon для GitHub Pages: ассеты теперь учитывают `BASE_URL` и не дают 404 в проде.

## 2026-04-23
- Анимированный золотой контур теперь используется везде, где есть золотой статус, чтобы лучшие состояния выглядели единообразно по всему приложению.
- На графике роста цвета переосмыслены: общий капитал стал зеленым, вложения — нейтральными, а под линиями появились мягкие полупрозрачные градиенты.
- В блоке `Если подождать еще` акценты добавлены выборочно: зеленый и золотой используются только там, где дополнительное время действительно дает заметный выигрыш.
- Анимация золотого контура замедлена и выровнена, чтобы убрать рывок на стыке цикла.
- Перелив золотого статуса усилен: контур стал ярче, толще и получил более заметные блики.
- Для золотого статуса добавлен мягкий анимированный переливающийся контур, чтобы лучшие сценарии выделялись заметнее без перегруза интерфейса.
- Палитра статусных карточек уточнена: фиолетовый убран, золотой и оранжевый сильнее разведены по тону и контрасту.
- В ключевые карточки результатов добавлены мягкие статусные акценты: золотисто-фиолетовый для сильного запаса, зеленый для устойчивого плана, оранжевый для зон внимания и красный для критичных разрывов.
- Кнопки действий в формах и инспекторах приведены к одному правилу: `Сбросить`, `Сохранить`, `Применить` и удаление перенесены в нижние action-зоны блоков.
- Скролл у левой панели возвращен внутрь карточки `Входные данные`, а расчет `max-height` исправлен на более точный вариант от `100dvh`, чтобы не появлялось ложное переполнение.
- Лишний скролл у панели ввода убран: ограничение высоты и прокрутка переносились с карточки формы на контейнер левой колонки, чтобы скролл появлялся только при реальном переполнении окна.
- У панели входных данных убран правый градиентный разделитель, а внутренний скролл сделан тоньше и спокойнее визуально.
- Для длинной левой панели ввода добавлен внутренний скролл по высоте окна на desktop, чтобы до нижних полей и кнопки сброса всегда можно было добраться.
- Для корректной работы `sticky` горизонтальное скрытие переписано с `overflow-x: hidden` на `overflow-x: clip` у `html/body`, чтобы не создавать лишний scroll-контейнер.
- Исправлена основная причина некорректного `sticky` у левой панели: убран `overflow-x: hidden` с `.app`, который мешал блоку закрепляться при прокрутке.
- Левая панель ввода временно переводилась в отдельную фиксированную desktop-колонку для проверки поведения при длинной прокрутке.
- Панель ввода переработана: обновлены типографика, компоновка и нижняя кнопка сброса.
- Layout калькулятора перестроен под живую работу: на десктопе форма стала левой рабочей колонкой, а на мобильном идет первой перед итогами и графиком.
- На странице калькулятора убрано переключение между режимами: результаты, график и план по годам теперь видны сразу, а левая колонка стала навигацией по секциям.
- Удален footer, а левая навигация переведена в фиксированную колонку на всю высоту окна на десктопе с адаптивным откатом на мобильных.
- Tailwind удален из проекта: `ui`-слой переписан обратно на Sass/CSS с тем же API компонентов, а Vite переведен на modern Sass API без предупреждения `legacy-js-api`.
- Проект переведен с локальных shadcn-like primitives на реальные `shadcn/ui`: подключены Tailwind/shadcn зависимости, официальный `ui`-слой и совместимость с текущими страницами.
- Добавлен общий footer с информацией о проекте, дисклеймером по расчетам, localStorage и ссылками на разделы.
- Подключен shadcn-style UI layer с локальными primitives и переписаны основные панели, поля и навигация на новые UI-компоненты.
