# InvestCalc

Демо: https://rqik.github.io/InvestCalc/

InvestCalc — финансовый калькулятор для планирования накоплений и пенсионных ориентиров. Приложение помогает оценить будущий капитал, нужный ежемесячный взнос, влияние инфляции и разные сценарии без лишней сложности.

## Возможности

- расчет итогового капитала к концу срока;
- оценка суммы в сегодняшних деньгах с учетом инфляции;
- расчет нужного ежемесячного взноса и требуемой доходности;
- график роста капитала по годам;
- годовой план накоплений в таблице;
- сравнение сценариев и сохранение вариантов в `localStorage`;
- отдельная страница пенсионного планирования;
- светлая и темная темы.

## Стек

- React 18
- TypeScript
- Vite
- SCSS Modules
- Recharts
- GitHub Pages

## Быстрый старт

```bash
npm install
npm run dev
```

После запуска Vite откроет локальный адрес приложения. Для production-сборки:

```bash
npm run build
```

## Скрипты

| Команда | Назначение |
| --- | --- |
| `npm run dev` | запуск dev-сервера |
| `npm run build` | TypeScript-проверка и production-сборка |
| `npm run preview` | локальный просмотр production-сборки |
| `npm run build:pages` | сборка для GitHub Pages |

## Структура

```text
src/
├── components/   # UI и feature-компоненты
├── constants/    # константы приложения
├── hooks/        # React-хуки
├── layouts/      # layout-обертки
├── pages/        # страницы calculator и retirement
├── styles/       # глобальные tokens, base, mixins
├── types/        # общие TypeScript-типы
└── utils/        # расчеты, форматирование и storage helpers
```

Компоненты оформлены по принципу `1 компонент = 1 папка` и используют локальные `*.module.scss`.

## Навигация

Приложение использует hash-навигацию:

- `#calculator` — калькулятор накоплений;
- `#retirement` — пенсионное планирование.

## Деплой

Проект рассчитан на публикацию через GitHub Pages. Workflow собирает приложение после пуша в основную ветку, если в настройках репозитория выбран источник `GitHub Actions`.

Адрес опубликованной версии:

## Данные

Сценарии сохраняются локально в браузере через `localStorage`. Они не синхронизируются между устройствами без backend или отдельного механизма экспорта.
